# Growth Calculator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Gratia-wide revenue calculator at `growth/index.html` that models analysts, rates, utilization, and HC ramp against an annual goal with monthly actuals tracking.

**Architecture:** Single self-contained HTML file (inline CSS + vanilla JS), matching `apollo/latam-captive.html` patterns. Four sections: goal bar, team table, global controls, monthly breakdown. Actuals persist to localStorage.

**Tech Stack:** HTML, CSS, vanilla JavaScript, DM Sans font, no build step.

**Spec:** `docs/superpowers/specs/2026-03-24-growth-calculator-design.md`

**Reference file:** `apollo/latam-captive.html` — use this as the pattern for CSS classes, color tokens, input styling, nav bar, and page layout. Reuse: nav-bar, page, h1, subtitle, divider, section-label, hc-input, level-table, range input, and color variables. Do NOT copy: deck-wrapper, slide-area, slide mechanics, slide-footer, goTo() keyboard navigation — this is a single scrollable page, not a multi-slide deck.

- [ ] **Step 3: Write JS for monthly forecast**

Add function `updateMonthly()` called from `updateAll()`:

```
var MONTHS = ['Jan','Feb','Mar','Apr','May','Jun','Jul','Aug','Sep','Oct','Nov','Dec'];
var COMPLETED_MONTHS = 3; // Jan-Mar hardcoded
var compensationOn = false;
```

Logic:
1. Get baseline HC and team mix from team table (`getTeamMix()`)
2. Get hiring pace from slider
3. For each forecast month (index `i` from 0 to 11):
   - If `i < COMPLETED_MONTHS`: use actuals if available (Task 4), else use forecast
   - For forecast months (`i >= COMPLETED_MONTHS`):
     - `monthsFromForecastStart = i - COMPLETED_MONTHS + 1` (first forecast month = 1, so April gets +hiringPace immediately)
     - `newHires = hiringPace × monthsFromForecastStart`
     - Distribute new hires proportionally across levels:
       ```
       totalBaseHC = sum of all level HCs from team table
       if totalBaseHC == 0: assign all newHires to A3
       else for each level:
         share = level.hc / totalBaseHC
         level.forecastHC = level.hc + Math.round(newHires * share)
       // Fix rounding: add/subtract remainder to largest-HC level
       ```
     - `totalMonthHC = sum of all level.forecastHC`
     - Revenue = sum of each level's `(forecastHC × rateHr × utilization × hours)`
   - Store month's HC and revenue for later use
4. Calculate targets:
   - If compensation OFF: every month gets `goal / 12`
   - If compensation ON: single-pass calculation — `remaining = max(0, goal - sum_of_completed_actuals)`, then `remaining / (12 - COMPLETED_MONTHS)` for each forecast month. If all 12 months completed, disable toggle.
5. Calculate gap per month: revenue - target (green if >=0, red if <0)
6. Render rows: HC, Utilization (global for forecast months), Revenue, Target, Gap
7. Update summary bar: annual goal, projected (sum of 12 months), gap, % of goal

Build header row with month labels. Completed months get highlighted background (`#e8f5f0`).

- [ ] **Step 3: Add localStorage persistence**

```javascript
var STORAGE_KEY = 'gratia-growth-calc-actuals';

function loadActuals() {
  try { return JSON.parse(localStorage.getItem(STORAGE_KEY)) || {}; }
  catch(e) { return {}; }
}

function saveActuals() {
  var data = {};
  for (var i = 0; i < COMPLETED_MONTHS; i++) {
    var hcInput = document.getElementById('actual-hc-' + i);
    var utilInput = document.getElementById('actual-util-' + i);
    var revInput = document.getElementById('actual-rev-' + i);
    if (hcInput && hcInput.value) data['hc-' + i] = +hcInput.value;
    if (utilInput && utilInput.value) data['util-' + i] = +utilInput.value;
    if (revInput && revInput.value) data['rev-' + i] = +revInput.value;
  }
  // Also save Apr HC override if present
  var aprHC = document.getElementById('actual-hc-3');
  if (aprHC && aprHC.value) data['hc-3'] = +aprHC.value;
  localStorage.setItem(STORAGE_KEY, JSON.stringify(data));
}
```

Modify `updateMonthly()` to call `loadActuals()` and populate inputs on render. Use actuals in calculations when present, forecast when not.
